# 赛博旅游 App 一期 MVP 执行方案

## 1. 文档目标

本文档将一期 MVP 从“产品想法”拆成可执行的工作包，方便你按阶段推进设计、内容、建模和开发。

## 2. 一期 MVP 定义

一期 MVP 只做一件事：

让用户能从全国地图进入城市，再进入景点详情，并对少量景点完成基础 3D 浏览。

一期不是为了做完所有功能，而是为了验证主链路、视觉方向和内容生产流程。

## 3. 一期 MVP 最终交付物

一期完成后，应至少包含以下结果：

- 1 个可点击的全国卡通地图首页
- 3 个开放城市
- 6 到 10 个景点
- 3 到 5 个可浏览 3D 景点
- 城市页和景点详情页
- 每个景点的标准化简介
- 一套后续可复用的内容生产规范

## 4. 工作拆解

### 4.1 产品规划包

交付内容：

- MVP PRD
- 版本路线图
- 页面信息架构
- 首批城市和景点清单

完成标准：

- 每个页面目标明确
- 每个模块的边界清楚
- 首版功能范围被严格限制

### 4.2 交互原型包

交付内容：

- 地图首页低保真原型
- 城市页低保真原型
- 景点详情页低保真原型

完成标准：

- 页面层级清楚
- 页面跳转顺畅
- 3 次点击内进入景点详情

### 4.3 视觉设定包

交付内容：

- 全国地图风格方向
- 城市页视觉风格
- 景点封面图风格样张
- UI 基础样式定义

完成标准：

- 视觉风格统一
- 景点辨识度高
- 城市和景点不会像素材拼盘

### 4.4 内容生产包

交付内容：

- 首批城市和景点文案
- 卡通景点封面图
- 城市头图
- 景点数据表

完成标准：

- 每个景点至少有封面图和简介
- 文案字段统一
- 资源命名清楚可管理

### 4.5 3D 资产包

交付内容：

- 3 到 5 个核心景点低模资源
- 统一输出格式
- 可用于移动端展示的压缩资源

完成标准：

- 可旋转
- 可缩放
- 加载性能可接受

### 4.6 技术开发包

交付内容：

- 地图页
- 城市页
- 景点页
- 3D 展示组件
- 内容配置结构

完成标准：

- 主链路打通
- 资源可替换和扩充
- 页面切换稳定

## 5. 建议排期

以下排期按轻量团队节奏编排，可按实际资源调整。

### 第 1 阶段：产品与范围确认

目标：
把要做什么和不做什么彻底定死。

任务：

- 确定首批 3 个城市
- 确定每个城市的景点列表
- 确定 MVP 页面结构
- 确定功能优先级

阶段产出：

- PRD
- 页面结构图
- 景点清单

### 第 2 阶段：原型与视觉方向

目标：
先把页面体验和风格方向跑通。

任务：

- 完成页面低保真原型
- 确定地图样式
- 确定景点卡通风格
- 输出 1 套视觉样张

阶段产出：

- 原型稿
- 风格稿

### 第 3 阶段：内容试制

目标：
验证“出图 -> 挑景点 -> 转 3D”的内容流程。

任务：

- 生成所有首批景点的卡通图
- 选出 3 到 5 个景点做 3D
- 为所有景点补齐基础文案

阶段产出：

- 首批图文资源
- 首批 3D 资源

### 第 4 阶段：开发实现

目标：
把页面和内容接起来，形成可用产品。

任务：

- 实现地图页
- 实现城市页
- 实现景点详情页
- 接入内容数据
- 接入 3D 模型浏览

阶段产出：

- 可运行 Demo

### 第 5 阶段：联调与优化

目标：
解决性能和体验问题，形成可测试版本。

任务：

- 优化图片和模型加载
- 检查页面切换
- 检查点击热区
- 检查文案和视觉一致性

阶段产出：

- MVP 测试版

## 6. 一期优先级排序

### P0：必须完成

- 地图 -> 城市 -> 景点主链路
- 统一卡通视觉
- 少量 3D 景点
- 标准化景点简介

### P1：有余力再做

- 收藏
- 最近浏览
- 推荐景点

### P2：延后到后续版本

- 音频讲解
- 内部漫游
- 热点点击
- 城市路线

## 7. 一期资源建议

如果是小团队或个人推进，建议按以下角色来思考工作：

- 产品：负责范围、信息架构、版本控制
- 视觉：负责地图、城市页、景点封面风格统一
- 内容：负责景点筛选和文案
- 3D：负责少量核心景点低模制作
- 开发：负责页面、数据接入、3D 浏览

如果实际只有你一个人推进，就按这个顺序做：

1. 定范围
2. 出原型
3. 定风格
4. 批量出图文
5. 试做少量 3D
6. 再开发页面

## 8. 一期上线判定

满足以下条件才算一期完成：

- 地图点击能稳定进入城市
- 城市页能稳定进入景点
- 景点页内容完整可浏览
- 至少 3 个景点 3D 可交互
- 视觉风格统一
- 内容结构可支持继续扩城市

## 9. 一期结束后立刻要做的事

一期上线后，不要立刻扩全国，先做复盘。

优先复盘这几个问题：

- 哪个城市点击最多
- 哪个景点停留最长
- 用户是否更喜欢图文还是 3D
- 内容生产最卡的是出图、建模还是接入

这一步决定后续是优先扩内容，还是优先补沉浸体验。
