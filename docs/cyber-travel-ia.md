# 赛博旅游 App 页面信息架构

## 1. 目标

定义 V0.1 MVP 的页面结构、导航关系和页面职责，确保后续设计和开发时不会偏离主链路。

## 2. 页面树

```text
App
├── 全国地图首页
├── 城市景点总览页
├── 景点详情页
└── 基础功能页
    ├── 收藏列表
    └── 最近浏览
```

## 3. 页面职责

### 3.1 全国地图首页

职责：

- 展示全国卡通地图
- 高亮开放城市
- 承接用户第一次点击

关键内容：

- App 标题
- 副标题
- 全国地图
- 城市点击点
- 推荐城市入口

### 3.2 城市景点总览页

职责：

- 展示城市主题
- 聚合该城市的景点
- 帮助用户快速选中一个景点

关键内容：

- 城市名称
- 城市标签
- 城市头图
- 景点卡片列表

### 3.3 景点详情页

职责：

- 展示景点视觉
- 展示基础介绍
- 提供 3D 浏览

关键内容：

- 景点名称
- 封面图
- 简介
- 3D 展示区
- 基础操作提示

### 3.4 收藏列表

职责：

- 保存用户感兴趣的景点

首版建议：

- 可延后实现
- 先做本地存储版本

### 3.5 最近浏览

职责：

- 方便用户回看刚看过的景点

首版建议：

- 可延后实现
- 只保留最近 20 条

## 4. 路由建议

```text
/                       全国地图首页
/city/:citySlug         城市景点总览页
/spot/:spotSlug         景点详情页
/favorites              收藏列表
/recent                 最近浏览
```

## 5. 跳转关系

```text
全国地图首页 -> 城市景点总览页 -> 景点详情页
景点详情页 -> 城市景点总览页
城市景点总览页 -> 全国地图首页
全国地图首页 -> 推荐城市 -> 城市景点总览页
```

## 6. 页面组件拆分

### 6.1 地图页组件

- `Header`
- `ChinaMap`
- `CityPin`
- `RecommendedCities`

### 6.2 城市页组件

- `CityHero`
- `CityTagline`
- `SpotCardList`
- `SpotCard`

### 6.3 景点页组件

- `SpotHero`
- `SpotSummary`
- `ModelViewer`
- `SpotTags`
- `RelatedSpots`

## 7. 首版交互原则

- 3 次点击内到达任意景点
- 每一层页面都能清楚返回上一层
- 未开放城市明确弱化
- 景点 3D 不可用时必须有静态替代图

## 8. 后续扩展位置

以下模块先预留位置，但不在 V0.1 强制实现：

- 搜索
- 导航地图缩放
- 热点讲解
- 景点内部漫游
- 音频导览
